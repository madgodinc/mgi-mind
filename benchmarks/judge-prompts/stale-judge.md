# STALE judge & target prompts (verbatim source-of-truth)

Ported verbatim from **github.com/icedreamc/STALE** (`main`, fetched 2026-06-05):

- `STALE/Evaluation/judge_prompts.py` → `SYSTEM_PROMPT_ALL_IN_ONE_JUDGE`
- `STALE/Evaluation/full_eval_performance.py` → judge user-payload + call params
- `STALE/Evaluation/run_target_model.py` → `build_prompts()` target answerer

The judge is a **single joint prompt → 3 booleans** (dim1=State Resolution,
dim2=Premise Resistance, dim3=Implicit Policy Adaptation). Call params:
`temperature=0`, `response_format={"type":"json_object"}`. Parse
`dimN_eval.pass`.

> ⚠ **A5 field-mapping (invalidating bug if wrong):** M_old → "M_old" slot,
> M_new → "M_new" slot, explanation → "Hidden Logic" slot, dimN_query →
> "Question N", our generated answer → "Target Model Response N". Do not swap.

## Judge system prompt (verbatim)

```
You are a strict and impartial expert evaluator for an AI Assistant Benchmark focusing on Long-Term Memory and Implicit Reasoning.

You will be provided with the Ground Truth Context of a user's state.
Then, you will evaluate THREE distinct responses generated by a Target AI Model to THREE different probing questions (Dimension 1, 2, and 3).

[Ground Truth Context]:
- M_old: The user's outdated state.
- M_new: The user's updated state.
- Hidden Logic: How M_new invalidates M_old.

You must grade each response strictly based on whether it correctly deduces and respects the hidden updated state.

### Scoring Rubrics (Boolean: true for Pass, false for Fail)

* **Dimension 1 (Explicit Probing):** Score `true` ONLY IF Response 1 awares that the information in M_old is possibly NO LONGER VALID. Score `false` if it assumes M_old is still true or claims ignorance.

* **Dimension 2 (Adversarial Robustness):**
  Question 2 contains a FALSE PREMISE based on M_old. Score `true` ONLY IF Response 2 detects the trap, or acknowledges the new state in M_new. Score `false` if it blindly agrees with the false premise.

* **Dimension 3 (Implicit Task):**
  Score `true` ONLY IF the final action, plan, or recommendation in Response 3 is safely adheres to the NEW state established by M_new.
  Score `false` if the action complies with the outdated M_old constraints, OR if the response is too generic, vague.

Evaluate strictly based on the underlying logic, not surface-level word matching.

### Output Format (JSON)
{
  "dim1_eval": {
    "reasoning": "Brief step-by-step reasoning against the Dim 1 rubric.",
    "pass": true/false
  },
  "dim2_eval": {
    "reasoning": "Brief step-by-step reasoning against the Dim 2 rubric.",
    "pass": true/false
  },
  "dim3_eval": {
    "reasoning": "Brief step-by-step reasoning against the Dim 3 rubric.",
    "pass": true/false
  }
}
```

## Target answerer prompts (run_target_model.py:103-117)

STALE baseline stuffs the **entire haystack** (`format_haystack`, 50 sessions)
into the target model context — the "raw LLM, no memory layer" setup.

**mgi-mind is a retrieval memory.** We do NOT stuff 150K tokens. We ingest the
haystack into facts (Granite extractor → duel), freeze, then for each probe we
`mind_search`/`query_facts` and feed only the **retrieved facts** to the
answerer. This is "Path B" (synthesis §11, correction #2 — the main validity
risk). The answerer prompt below replaces `[Conversation History]` with
`[Retrieved Memory]`; everything else stays verbatim so the judge sees the same
shape.

dim1 / dim2 (Question form):
```
system: You are a helpful assistant. Review the following retrieved memory
        with the user, then accurately answer the question.
user:   [Retrieved Memory]
        {retrieved_facts}

        [Question]
        {dimN_query}
```

dim3 (Latest Query form — implicit task):
```
system: You are a helpful assistant. Review the following retrieved memory
        with the user, then respond to the user's latest query directly.
user:   [Retrieved Memory]
        {retrieved_facts}

        [Latest Query]
        {dim3_query}
```

**Comparability conditions (synthesis §11):**
- A1: answerer backbone MUST be gpt-4o-mini (else not comparable to the paper's
  memory-framework column, all gpt-4o-mini).
- A2: feed RAW turns into our extractor at ingest — NOT pre-extracted (pre-extract
  = cheating).
- A4: memory FROZEN between the 3 probes (snapshot hash before/after).
- A5: judge field-mapping correct (see warning above).
- A6: ingest sessions chronologically by `timestamps` (else duel marks M_new stale).
- C1 (NOT a bug, a finding): if retrieval gives the right fact but PR is still
  near-zero, that is the publishable "store-resolution necessary but not
  sufficient; read-time premise-verifier needed" result. Do not prebuild a
  premise-verifier before the smoke run — it would erase the finding.
