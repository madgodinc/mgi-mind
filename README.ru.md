# MGI-Mind

**[English](README.md)** | **[Русский](README.ru.md)** | **[中文](README.zh.md)**

Долговременная память для AI-ассистентов, на самохостинге. Ассистент сам сохраняет
важное по ходу работы и потом находит это по смыслу - перестаёт переспрашивать одно и
то же и не начинает каждую сессию с чистого листа. Всё крутится на твоей машине: один
бинарь на Rust, локальная векторная база Qdrant и локальные ONNX-модели. Без облака,
без API-ключей, данные никуда не уходят.

```
Ты:  какой там был адрес сервера для деплоя?

Ассистент (вызывает mind_search "сервер деплоя"):
  -> "Деплой-сервер 10.0.0.5:8080, SSH под deploy@, ключ в vault"  (source: infra.md)

Ты:  о, спасибо
```

Подключается к ассистенту по MCP (Model Context Protocol), так что Claude Code и
похожие инструменты читают и пишут память сами. Плюс это обычный CLI - можно руками.

> **Если у тебя контент в основном на русском** - поставь `rerank_enabled = false` в
> `~/mgimind/config.json`. Дефолтный реранкер заточен под английский и русскую
> выдачу ухудшает; без него гибридный поиск ранжирует русский хорошо. Подробнее в
> разделе [Языки и реранкер](#языки-и-реранкер).

## Что это

MGI-Mind - это слой памяти между тобой и ассистентом. Ассистент по ходу разговора
пишет короткие заметки («воспоминания») и факты, а потом достаёт нужные, когда они к
месту. Ты ничего не тегаешь, не раскладываешь по папкам.

Сделано ради качества поиска, а не просто хранения:

- **Гибридный поиск.** Каждое воспоминание хранится сразу двумя векторами: плотный
  (dense, multilingual-e5-base) ловит смысл, разреженный частотный (TF-IDF,
  BM25-подобный) ловит точные слова. Запрос идёт по обоим, два ранжированных списка
  сливаются через Reciprocal Rank Fusion. В итоге и смысловая полнота («серверная
  коробка» находит «деплой-хост»), и точность по словам («fossilize_replay» находит
  ровно ту заметку, где это слово есть) - за один запрос.
- **Реранкер (cross-encoder).** Топ кандидатов после слияния пере-оценивает
  cross-encoder (bge-reranker-base), который читает запрос и текст вместе - это сильно
  точнее, чем сравнивать вектора. Включён по умолчанию, силён на английском
  (про другие языки - ниже в разделе [Языки и реранкер](#языки-и-реранкер)).
- **Один тёплый процесс.** Сервер `mgimind mcp` сам и есть MCP-сервер: он живёт всю
  сессию, поэтому модели грузятся один раз и остаются тёплыми в памяти, а поиск стоит
  миллисекунды, а не перезагрузку модели на каждый вызов. Никакого демона, никакого
  сокета, никакой Node-обёртки - один бинарь на Rust, говорящий по MCP через stdio.

Вокруг поиска есть небольшой граф знаний для структурированных фактов, посессионные
логи для непрерывности и зашифрованное хранилище секретов (только из терминала).

## Зачем

Ассистент без памяти повторяется, снова просит тот же контекст и не может опереться на
вчерашнюю работу. Обычный костыль - ты сам ведёшь заметки, теги, папки; это бесконечная
рутина, и ассистент всё равно толком их не читает.

У альтернатив есть реальные дыры:

- **Обычные заметки (Obsidian, Notion).** Хороши для тебя, но ассистент не ищет в них
  по смыслу, а раскладываешь всё ты.
- **Голая векторная база.** Даёт семантику, но без точного совпадения слов, без
  реранка, без дедупа, без сессий, без фактов, без работы с секретами - всё это
  собирай сам.
- **Облачное «memory»-API.** Отправляет твои данные на чужие серверы.

MGI-Mind - это собранная локальная версия: гибридный поиск с реранком, дедуп, факты,
сессии и vault за одним бинарём, который ты запускаешь сам. Данные твои и не покидают
машину.

## Быстрый старт

Одна команда. Установщик качает бинарь под твою ОС, кладёт его в PATH и сам
запускает `init` + `doctor --fix` (это вытянет Qdrant, ONNX Runtime и модели).

**Linux / macOS:**

```bash
curl -fsSL https://raw.githubusercontent.com/madgodinc/mgi-mind/main/install.sh | sh
```

**Windows** (PowerShell):

```powershell
irm https://raw.githubusercontent.com/madgodinc/mgi-mind/main/install.ps1 | iex
```

В конце он печатает готовую команду для подключения к Claude Code:

```bash
claude mcp add mgimind -- /home/you/.local/bin/mgimind mcp
```

Готово. `mgimind mcp` И ЕСТЬ MCP-сервер; он живёт всю сессию с тёплыми моделями и
сам поднимает встроенный Qdrant при первом обращении - отдельный сервис не нужен.
Один раз покажи ассистенту [`AI_INSTRUCTIONS.md`](AI_INSTRUCTIONS.md), чтобы он знал
протокол (логировать сессию, искать перед ответом, секреты - в vault).

`doctor --fix` кладёт в `~/mgimind/`: ONNX Runtime (библиотеку, которую грузит
эмбеддер), бинарь Qdrant, модель эмбеддингов (multilingual-e5-base, квантованный ONNX,
~270 МБ) и реранкер (bge-reranker-base, ~280 МБ).

### Параметры установщика

- `INSTALL_DIR=/opt/mgimind curl ... | sh` - поставить не в `~/.local/bin`, а куда укажешь.
- `MGIMIND_TAG=v0.8.0 curl ... | sh` - зафиксировать конкретный релиз вместо `latest`.
- `SKIP_DOCTOR=1 curl ... | sh` - только положить бинарь; `init` + `doctor --fix` запустишь сам.

### Установка вручную (без установщика)

Если не хочешь скармливать sh-скрипт в шелл - возьми архив релиза под свою ОС из
[Releases](https://github.com/madgodinc/mgi-mind/releases/latest), положи `mgimind` в
PATH, потом:

```bash
mgimind init
mgimind doctor --fix
claude mcp add mgimind -- /абсолютный/путь/mgimind mcp
```

**Проба из CLI** (по желанию - тот же бинарь работает как обычный инструмент командной
строки):

```bash
mgimind create work
mgimind add work "Деплой-сервер 10.0.0.5:8080, SSH под deploy@"
mgimind search "как достучаться до деплой-сервера"
```

### Заметки по ОС

- **Linux x86_64, macOS arm64 (Apple Silicon), Windows x86_64.** Готовые бинари
  есть в каждом релизе. Установщик сам выбирает нужный.
- **macOS Intel (x86_64).** Готового бинаря нет. Хостинговый `macos-13` runner у
  GitHub стоит в очереди по 20-30+ минут и постепенно выводится из эксплуатации,
  поэтому он убран из матрицы релиза. Собирай из исходников (следующий раздел) -
  это пара минут.
- **Карантин macOS при первом запуске.** Скачанному бинарю может понадобиться
  `xattr -d com.apple.quarantine /путь/к/mgimind` либо правый клик -> Open один раз в Finder.
- **Windows.** SmartScreen может предупредить о неподписанном `mgimind.exe` («Windows
  protected your PC») - нажми **More info -> Run anyway**. Антивирус тоже может поместить
  бинарь или скачанные модели в карантин; если `mgimind doctor` пишет, что файл скачан,
  но его нет, разреши `mgimind.exe` и папку `%USERPROFILE%\mgimind` в антивирусе и
  повтори `mgimind doctor --fix`. (Это GUI-окна, терминал по ним не кликнет; подпись кода,
  которая уберёт предупреждение SmartScreen, - в планах.)

### Сборка из исходников

Если под твою платформу нет релиза или хочешь собрать сам, нужен тулчейн Rust (`rustup`);
других зависимостей нет.

```bash
git clone https://github.com/madgodinc/mgi-mind.git
cd mgi-mind
cargo build --release                  # бинарь: target/release/mgimind
```

Затем `target/release/mgimind init && target/release/mgimind doctor --fix` и подключи
через `claude mcp add mgimind -- /абсолютный/путь/target/release/mgimind mcp`.

## Как пользоваться

**Руками (CLI).**

```bash
mgimind add notes "День рождения мамы 14 марта, любит пионы" --source personal
mgimind search "когда у мамы день рождения"
#  1. [notes] (score: 0.94) День рождения мамы 14 марта, любит пионы
#     source: personal

mgimind fact add "пользователь" "предпочитает" "Rust"
mgimind fact query "пользователь"
#   пользователь -> предпочитает -> Rust

mgimind history --limit 3              # три последних воспоминания
mgimind stats                          # счётчики по библиотекам, фактам, сессиям
```

**Через ассистента (MCP).** После подключения просто говоришь - инструменты он
вызывает сам:

```
Ты:  запомни, пароль от стейджинг-БД лежит в vault под ключом "staging-db"
Ассистент: (mind_add) записал. Сам секрет остаётся в твоём терминальном vault, не здесь.

Ты:  какая база на стейджинге?
Ассистент: (mind_search "стейджинг база") Postgres 16, хост db-staging.internal:5432
```

Поиск отдаёт результат тирами, чтобы экономить токены: `--tier 1` - сниппет ~100
символов, `--tier 2` (по умолчанию) - ~500, `--tier 3` - полный текст.

## Как устроено

```
  твой AI-ассистент
        |  MCP (JSON-RPC через stdio)
        v
  mgimind mcp  (один процесс на Rust: MCP-сервер + эмбеддер, модели тёплые)
        |  поднимается при первом обращении
        v
  Qdrant (локально, только loopback)
        |
        одна коллекция "memories", два вектора на точку:
        dense (e5, смысл) + sparse (TF-IDF, точные слова)
```

**Хранилище.** Все воспоминания в одной коллекции Qdrant. Поле `library` разделяет
пространства имён (работа, личное, проект), а запрос может фильтроваться по одной
библиотеке или искать по всем. ID точки - UUIDv5 от `library + content`, поэтому
повторное добавление того же текста просто перезаписывает ту же точку: ни дублей, ни
гонок. Datetime-индекс на `created_at` позволяет `history` сразу брать новейшие N, не
вычитывая всё.

**Эмбеддинги.** Текст кодируется локально через ONNX Runtime. По умолчанию
multilingual-e5-base (768 измерений) - силён на английском, тянет смешанные языки.
Эмбеддер настраиваемый: пулинг (mean или CLS), вход `token_type_ids` и префиксы
query/passage - всё в конфиге, смена модели не требует правок кода. Вход обрезается до
512 токенов, а `add` режет длинный текст на чанки, чтобы ничего за пределом не
терялось молча.

**Поиск.** Запрос кодируется один раз. Qdrant за один вызов Query API делает плотный
поиск ближайших и разреженный поиск и сливает их через RRF. Если реранк включён, топ
`rerank_top_k` пере-оценивается cross-encoder'ом и переупорядочивается. Фильтр по
`library`, если задан, применяется к обеим веткам.

**Безопасность.** Загрузки проверяются по зашитым SHA-256, fail-closed. Qdrant слушает
только loopback и может требовать API-ключ. Vault - только из терминала. Запись файлов
атомарна (temp, fsync, rename, fsync каталога), так что краш оставит либо старый файл,
либо новый, но не битый.

## Справочник команд

### Память

| Команда | Что делает |
|---|---|
| `mgimind add <lib> <текст> [--source <тег>]` | Сохранить воспоминание. Длинный текст чанкуется; печатает число чанков. |
| `mgimind search <запрос> [--library <l>] [--limit N] [--tier 1\|2\|3]` | Гибридный поиск + реранк. Тир задаёт объём текста в ответе. |
| `mgimind history [--limit N]` | Последние воспоминания, новейшие сверху. |
| `mgimind delete <lib> <id>` | Удалить одно воспоминание по id (id виден в выдаче). |
| `mgimind context` | Краткий брифинг к началу сессии: прошлая сессия, свежие факты, библиотеки. |

### Библиотеки

| Команда | Что делает |
|---|---|
| `mgimind create <name>` | Зарегистрировать библиотеку. |
| `mgimind list` | Список библиотек. |
| `mgimind drop <name>` | Удалить библиотеку и все её воспоминания. |
| `mgimind stats` | Счётчики по библиотекам, фактам, сессиям, состояние vault. |

### Граф знаний

| Команда | Что делает |
|---|---|
| `mgimind fact add <s> <p> <o>` | Сохранить факт-тройку. Та же тройка перезаписывается (дедуп). |
| `mgimind fact query <term>` | Найти факты по совпадению в субъекте/предикате/объекте. |
| `mgimind fact invalidate <id>` | Мягкое удаление факта (остаётся на диске, помечен невалидным, скрыт из выдачи). |

### Сессии

| Команда | Что делает |
|---|---|
| `mgimind session start --agent <имя>` | Начать лог сессии для агента. |
| `mgimind session end --agent <имя> --summary <текст>` | Завершить с итогом. |
| `mgimind session last [--agent <имя>]` | Показать последнюю сессию (можно по агенту). |

### Vault (только терминал)

| Команда | Что делает |
|---|---|
| `mgimind vault store <k> <v> [--category c] [--desc d]` | Сохранить зашифрованный секрет. |
| `mgimind vault get <k>` | Получить секрет (спросит мастер-пароль и подтверждение). |
| `mgimind vault list` | Список ключей (значения не показываются). |
| `mgimind vault delete <k>` | Удалить секрет. |

### Сервис и данные

| Команда | Что делает |
|---|---|
| `mgimind mcp` | Запуск как MCP-сервер по stdio (к нему подключается ассистент). Один тёплый процесс; Qdrant стартует сам. |
| `mgimind serve` / `mgimind stop` | Запуск / остановка встроенного Qdrant вручную (обычно не нужно - `mcp` делает это сам). |
| `mgimind migrate [--purge]` | Переэмбеддить старые по-библиотечные коллекции в одну `memories`. Идемпотентно. `--purge` удаляет старые после. |
| `mgimind backup <file>` / `mgimind restore <file>` | gzip+tar всего каталога данных. |
| `mgimind export [--format json\|md] [--output <dir>]` | Экспорт воспоминаний в файлы. |
| `mgimind import <obsidian\|markdown> <path> [--library <l>]` | Импорт папки markdown (рекурсивно, с чанкингом). |
| `mgimind doctor [--fix]` | Проверка; `--fix` докачивает недостающее. |

## Настройки

Конфиг - `~/mgimind/config.json`. Поля, влияющие на поиск:

| Поле | По умолчанию | Что значит |
|---|---|---|
| `model_name` | `multilingual-e5-base` | Папка модели в `models/`. |
| `vector_size` | `768` | Размерность эмбеддинга (должна совпадать с моделью). |
| `pooling` | `mean` | `mean` (e5, MiniLM) или `cls` (часть XLM-R). |
| `uses_token_type_ids` | `false` | `true` для BERT-семейства, `false` для XLM-R / e5. |
| `query_prefix` / `passage_prefix` | `query: ` / `passage: ` | Для e5 нужны, для других пусто. |
| `rerank_enabled` | `true` | Реранк cross-encoder. Смотри раздел про языки. |
| `rerank_model` | `bge-reranker-base` | Папка реранкера в `models/`. |
| `rerank_top_k` | `20` | Сколько кандидатов брать и реранкать перед выдачей `limit`. |
| `qdrant_port` | `6334` | gRPC-порт Qdrant. |
| `qdrant_api_key` | нет | Если задан, Qdrant стартует с ним, клиент аутентифицируется. |

## Языки и реранкер

Дефолтная связка заточена под английский - потому что сам ассистент лучше всего
рассуждает на нём и модели на нём сильнее. Для англоязычного проекта это рекомендуемый
вариант.

Несколько честных деталей:

- Эмбеддер (multilingual-e5-base) реально мультиязычный. Английский запрос находит
  русскую заметку и наоборот; один только поиск работает между языками хорошо.
- Дефолтный реранкер (bge-reranker-base) заточен под английский. Английскую выдачу он
  улучшает, но **русскую ранжировку ухудшает** (как и другие неанглийские). Это
  единственное место, где дефолты в пользу английского.
- **Если контент в основном русский (или другой неанглийский):** поставь
  `rerank_enabled = false`. Гибридный dense+sparse сам ранжирует такие языки хорошо;
  ухудшает именно англо-заточенный реранкер. Либо подставь сильный мультиязычный
  реранкер.
- Реранк стоит инференса cross-encoder на запрос: примерно 1-2 секунды на 20
  кандидатов на CPU. Снизь `rerank_top_k` или выключи реранк ради скорости.

## Смена модели эмбеддингов

Смена модели обычно меняет размерность, поэтому воспоминания надо переэмбеддить:

1. Сначала бэкап: `mgimind backup ~/mgi-backup.tar.gz`.
2. Пропиши в `config.json` под новую модель: `model_name`, `vector_size`, `pooling`,
   `uses_token_type_ids`, префиксы.
3. `mgimind doctor --fix` - скачает её. (Запинены только встроенные дефолты; кастомная
   модель качается с предупреждением «integrity not verified» - запинь её SHA-256 в
   `integrity.rs`, если важно.)
4. `mgimind migrate` - переэмбеддит всё из сохранённого текста под новую модель.

## Если что-то не работает

- **«Model not found ... run doctor --fix».** Модели нет в `~/mgimind/models/`.
  Запусти `mgimind doctor --fix`.
- **«invalid expand shape» / ошибки инференса.** Обычно вход сильно больше 512
  токенов. `add` чанкует сам; если зовёшь библиотеку напрямую - чанкуй заранее.
- **Поиск медленный.** Это реранк на CPU. Снизь `rerank_top_k` или поставь
  `rerank_enabled = false`. (Модели остаются тёплыми сами по себе на всё время жизни
  процесса `mgimind mcp`, так что цену загрузки платит только первый поиск за сессию.)
- **Тул падает сразу после установки.** Запусти `mgimind doctor` (ассистент может
  вызвать `mind_doctor`): он точно скажет, чего не хватает - Qdrant не запущен, модель
  не скачана, нет ONNX Runtime или файл в карантине антивируса - а `--fix` докачает,
  что сможет.
- **Предупреждение о несовпадении размерности на `serve`.** Вектора коллекции не
  совпали с `vector_size`, обычно после смены модели. Переэмбедди через
  `mgimind migrate`.
- **Русская выдача мимо.** Поставь `rerank_enabled = false` (см. раздел про языки).

## Безопасность

- Загрузки проверяются по зашитым SHA-256 (fail-closed) для ONNX Runtime (linux-x64),
  Qdrant и дефолтных моделей (e5 и реранкер). Другие платформы и кастомные модели -
  с предупреждением, без слепого доверия.
- Qdrant слушает только `127.0.0.1`, поддерживает API-ключ.
- Vault - это AES-256-GCM с ключом, выведенным через Argon2id (параметры зашиты, чтобы
  апгрейд крейта не заблокировал доступ). Только из терминала: мастер-пароль и
  расшифрованные секреты не ходят через MCP. MCP-тулы `mind_vault_*` возвращают
  инструкции для терминала, но никогда сам секрет.
- Запись файлов атомарна и с fsync каталога - после краша остаётся старый или новый
  файл, не битый.
