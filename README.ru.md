# MGI-Mind

**[English](README.md)** | **[Русский](README.ru.md)** | **[中文](README.zh.md)**

**[Последний релиз: v1.6.4](https://github.com/madgodinc/mgi-mind/releases/tag/v1.6.4)** · **[CHANGELOG](CHANGELOG.md)** · **[Discussions](https://github.com/madgodinc/mgi-mind/discussions)** · **[Issues](https://github.com/madgodinc/mgi-mind/issues)** · **[Contributing](CONTRIBUTING.md)**

Локальная долговременная память для ИИ-ассистентов. Один бинарник на Rust,
локальная векторная БД Qdrant, локальные ONNX-модели. Говорит по MCP, так
что Claude Code и другие ассистенты читают и пишут память сами. Также
обычный CLI.

```
You:  на каком сервере у нас деплой?

Assistant (вызывает mind_search "deploy server"):
  -> "Deploy server is 10.0.0.5:8080, SSH as deploy@, key in vault"  (source: infra.md)

You:  спасибо
```

Ничего не уходит с машины — эмбеддинги, поиск, реранкинг, vault всё
локально. Никаких облачных аккаунтов, API-ключей, телеметрии.

## Что это

MGI-Mind сидит между тобой и твоим ассистентом. Ассистент пишет короткие
заметки («памяти») и факты по ходу разговора, и достаёт релевантные
обратно, когда они нужны.

Поиск, не просто хранение:

- **Гибридный поиск.** Каждая память хранится как два вектора: плотный
  (multilingual-e5-base, 768 измерений) для смысла, и разреженный
  частотный (TF-IDF, в стиле BM25) для точных слов. Запрос пускает оба
  плеча и сливает их через Reciprocal Rank Fusion. «Серверная коробка»
  находит «deploy host» через плотное плечо; `fossilize_replay` находит
  одну заметку с этим токеном через разреженное.
- **Реранкинг кросс-энкодером.** Топ кандидатов после фузии
  переоценивается bge-reranker-base, который читает запрос и пассаж
  вместе — точнее, чем сравнение векторов. По умолчанию включено,
  заточено под английский — см. [Языки и реранкер](#языки-и-реранкер)
  про компромисс на других языках.
- **Один тёплый процесс.** `mgimind mcp` сам по себе и есть MCP-сервер:
  работает всю сессию, модели грузятся один раз и остаются в памяти,
  так что лукап стоит миллисекунды, а не перезагрузку модели на каждый
  вызов.

Вокруг поиска есть граф знаний для структурированных фактов, сессионные
логи на ассистента для сквозной непрерывности, и зашифрованный
терминальный vault для секретов.

## Зачем оно

Ассистент без памяти просит контекст каждую сессию и не может строить на
вчерашней работе. Обычный обход — ты сам ведёшь заметки, теги и папки,
которые ассистент всё равно не умеет читать по смыслу.

То, что MGI-Mind делает, а Obsidian и Notion не делают: **система сама
решает что записать, не ты.** MCP-сервер читает что ассистент делает в
реальном времени и маршрутизирует факты, решения и фиксы в хранилище
через гейт релевантности. Ты не складываешь, не теггируешь, не решаешь
что «стоит сохранить». Малозначные кандидаты падают в карантин
(восстанавливаются при повторном утверждении), не загрязняя поиск.

Как это соотносится с очевидными альтернативами:

- **Обычные заметки (Obsidian, Notion).** Сильны как личный блокнот, но
  ассистент не ищет в них по смыслу, и каждое нажатие клавиши одинаково
  важно для папки `.md`-файлов.
- **Голая векторная БД.** Семантический поиск, но нет точного
  совпадения, нет реранкинга, нет дедупликации, нет сессий, нет фактов,
  нет работы с секретами, нет гейта релевантности, нет процедурной
  памяти. Всё это собирается руками.
- **Хостинг-API памяти.** Твои данные на чьих-то серверах. Закрывает
  ящик от инспекции, от поведения дедупликации, от того что реально
  делает гейт релевантности.

MGI-Mind — собранная локальная версия: гибрид + реранк, гейт
релевантности, дедуп, факты, сессии, процедурная память (плейбуки
«ошибка → фикс»), и терминальный vault — за одним бинарником, который ты
запускаешь сам.

## Быстрый старт

Одна команда. Инсталлятор кладёт бинарник в PATH и запускает `init` +
`doctor --fix` (который качает Qdrant, ONNX Runtime и модели).

**Linux / macOS:**

```bash
curl -fsSL https://raw.githubusercontent.com/madgodinc/mgi-mind/main/install.sh | sh
```

**Windows** (PowerShell):

```powershell
irm https://raw.githubusercontent.com/madgodinc/mgi-mind/main/install.ps1 | iex
```

Когда установка закончится, напечатанная команда подключит сервер к
Claude Code:

```bash
claude mcp add mgimind -- /home/you/.local/bin/mgimind mcp
```

`mgimind mcp` И ЕСТЬ MCP-сервер; он работает всю сессию с тёплыми
моделями и поднимает встроенный Qdrant при первом использовании. Покажи
ассистенту [`AI_INSTRUCTIONS.md`](AI_INSTRUCTIONS.md) один раз, чтобы он
знал протокол (логировать сессию, искать перед ответом, использовать
vault для секретов).

`doctor --fix` качает в `~/mgimind/`: ONNX Runtime (библиотеку, которую
грузит эмбеддер), бинарник Qdrant, модель эмбеддингов
(multilingual-e5-base, квантованный ONNX, ~270 МБ) и реранкер
(bge-reranker-base, квантованный ONNX, ~280 МБ).

### Флаги инсталлятора

- `INSTALL_DIR=/opt/mgimind curl ... | sh` — поставить не в `~/.local/bin`.
- `MGIMIND_TAG=v1.0.1 curl ... | sh` — закрепить релиз вместо `latest`.
- `SKIP_DOCTOR=1 curl ... | sh` — только положить бинарник; `init` + `doctor --fix` потом сам.

### Ручная установка (без скрипта)

Если не хочется пайпать скрипт в shell, возьми архив релиза под свою ОС
из [Releases](https://github.com/madgodinc/mgi-mind/releases/latest),
положи `mgimind` в PATH, потом:

```bash
mgimind init
mgimind doctor --fix
claude mcp add mgimind -- /absolute/path/to/mgimind mcp
```

Попробовать из CLI (тот же бинарник — обычная команда):

```bash
mgimind create work
mgimind add work "Deploy server is 10.0.0.5:8080, SSH as deploy@"
mgimind search "как достучаться до deploy-сервера"
```

### Заметки по платформам

- **Linux x86_64, macOS arm64 (Apple Silicon), Windows x86_64** —
  собранные бинарники в каждом релизе. Инсталлятор берёт правильный.
- **macOS Intel (x86_64)** — собранного бинарника нет. Хостинговый
  раннер `macos-13` от GitHub сидит в очереди по 20-30+ минут и
  выводится из употребления, так что он исключён из релизной матрицы.
  Собирай из исходников (следующая секция); занимает несколько минут.
- **Карантин при первом запуске macOS** — скачанному бинарнику может
  понадобиться `xattr -d com.apple.quarantine /path/to/mgimind`, или
  правый клик → Open один раз в Finder.
- **Windows** — SmartScreen может ругаться на неподписанный
  `mgimind.exe` («Windows protected your PC») — выбери **More info →
  Run anyway**. Антивирус может также карантинить бинарник или модели,
  которые он качает; если `mgimind doctor` пишет, что файл скачан, но
  отсутствует — разреши `mgimind.exe` и папку `%USERPROFILE%\mgimind`
  в AV, потом запусти `mgimind doctor --fix` ещё раз. Подписание кода,
  чтобы убрать SmartScreen-промпт, в roadmap.

### Сборка из исходников

Rust-тулчейн (`rustup`); других зависимостей нет.

```bash
git clone https://github.com/madgodinc/mgi-mind.git
cd mgi-mind
cargo build --release                  # бинарник: target/release/mgimind
```

Потом `target/release/mgimind init && target/release/mgimind doctor --fix`
и подключение через
`claude mcp add mgimind -- /absolute/path/to/target/release/mgimind mcp`.

## Использование

**Из CLI:**

```bash
mgimind add notes "День рождения мамы 14 марта, она любит пионы" --source personal
mgimind search "когда у мамы день рождения"
#  1. [notes] (score: 0.94) День рождения мамы 14 марта, она любит пионы
#     source: personal

mgimind fact add "user" "prefers" "Rust"
mgimind fact query "user"
#   user -> prefers -> Rust

mgimind history --limit 3              # три самые свежие памяти
mgimind stats                          # счётчики по библиотекам, фактам, сессиям
```

**Через ассистента (MCP).** После подключения — просто говоришь.
Ассистент сам зовёт инструменты:

```
You:  запомни, что пароль от staging БД лежит в vault под "staging-db"
Assistant: (mind_add) сохранил. Сам секрет остаётся в твоём терминальном vault, не здесь.

You:  какая база у нас на staging?
Assistant: (mind_search "staging database") Postgres 16, host db-staging.internal:5432
```

Поиск возвращает результаты ярусами, чтобы ассистент тратил токены
аккуратно: `--tier 1` — фрагмент ~100 символов, `--tier 2` (по
умолчанию) — ~500, `--tier 3` — полный текст.

## Как устроено

```
  твой ИИ-ассистент
        |  MCP (JSON-RPC через stdio)
        v
  mgimind mcp  (один Rust-процесс: MCP-сервер + эмбеддер, модели тёплые)
        |  стартует при первом использовании
        v
  Qdrant (локальный, только loopback)
        |
        одна коллекция "memories", два вектора на точку:
        плотный (e5, смысл) + разреженный (TF-IDF, точные термы)
```

**Хранилище.** Все памяти живут в одной коллекции Qdrant. Поле `library`
на каждой точке разделяет неймспейсы (work, personal, проект), и запрос
может фильтровать по одной библиотеке или искать по всем. ID точки — это
UUIDv5 от `library + content`, так что добавление того же текста дважды
просто перезаписывает ту же точку — нет дубликатов, нет гонок. Индекс
`created_at` по datetime даёт `history` возвращать N свежих напрямую без
скана всего.

**Эмбеддинги.** Текст эмбеддится локально через ONNX Runtime. По
умолчанию multilingual-e5-base (768 измерений), силён на английском и
держит смешанные языки. Эмбеддер модельно-зависим: пулинг (mean или
CLS), вход `token_type_ids`, префиксы query/passage — всё конфиг, так
что смена модели не требует изменений кода. Входы режутся на 512
токенов; `add` дробит длинный текст на куски, чтобы ничего за пределами
cap'а не пропадало молча.

**Поиск.** Запрос эмбеддится один раз. Qdrant пускает плотный ANN-поиск
и разреженный поиск в одном вызове Query API и сливает их через RRF.
Если реранкинг включён, топ `rerank_top_k` кандидатов переоцениваются
кросс-энкодером и переупорядочиваются. Фильтр `library`, если задан,
применяется к обоим плечам.

**Безопасность.** Загрузки проверяются на pinned SHA-256 (fail-closed).
Qdrant биндится только на loopback и может требовать API-ключ. Vault
только терминальный — мастер-пароль и расшифрованные секреты никогда не
идут по MCP-каналу. Записи файлов атомарны (temp-файл, fsync, rename,
fsync директории), так что краш оставляет либо старый файл, либо новый,
никогда не битый.

## Справочник команд

### Память

| Команда | Что делает |
|---|---|
| `mgimind add <library> <content> [--source <tag>]` | Сохранить память. Длинный текст дробится; печатает, сколько кусков сохранено. |
| `mgimind search <query> [--library <l>] [--limit N] [--tier 1\|2\|3]` | Гибридный поиск, потом реранк. Tier задаёт, сколько текста возвращать. |
| `mgimind history [--limit N]` | Самые свежие памяти, новые первыми. |
| `mgimind delete <library> <id>` | Удалить одну память по id (id показывается в результатах поиска). |
| `mgimind context` | Компактный брифинг на старт сессии: последняя сессия, свежие факты, библиотеки. |

### Библиотеки

| Команда | Что делает |
|---|---|
| `mgimind create <name>` | Зарегистрировать библиотеку. |
| `mgimind list` | Список библиотек. |
| `mgimind drop <name>` | Удалить библиотеку и все её памяти. |
| `mgimind stats` | Счётчики по библиотекам, фактам, сессиям, состояние vault. |

### Граф знаний

| Команда | Что делает |
|---|---|
| `mgimind fact add <subject> <predicate> <object>` | Сохранить тройку-факт. Та же тройка перезаписывает (дедуп). |
| `mgimind fact query <term>` | Найти факты, где term встречается в subject, predicate или object. |
| `mgimind fact invalidate <id>` | Мягкое удаление факта (остаётся на диске, помечен невалидным, скрыт из запросов). |

### Сессии

| Команда | Что делает |
|---|---|
| `mgimind session start --agent <name>` | Начать сессионный лог для ассистента. |
| `mgimind session end --agent <name> --summary <text>` | Закрыть с резюме. |
| `mgimind session last [--agent <name>]` | Показать последнюю сессию (опционально для одного ассистента). |

### Vault (только терминал)

| Команда | Что делает |
|---|---|
| `mgimind vault store <key> <value> [--category c] [--desc d]` | Сохранить зашифрованный секрет. |
| `mgimind vault get <key>` | Получить секрет (просит мастер-пароль, потом подтверждение). |
| `mgimind vault list` | Список имён ключей (значения никогда не показываются). |
| `mgimind vault delete <key>` | Удалить секрет. |

### Сервис и данные

| Команда | Что делает |
|---|---|
| `mgimind mcp` | Запустить как MCP-сервер по stdio (то, к чему подключается ассистент). Один тёплый процесс; стартует Qdrant сам. |
| `mgimind serve` / `mgimind stop` | Старт/стоп встроенного Qdrant вручную (редко нужно — `mcp` делает это сам). |
| `mgimind migrate [--purge]` | Переэмбеддить старые библиотечно-разделённые коллекции в единую `memories`. Идемпотентно. `--purge` удаляет старые коллекции после. |
| `mgimind backup <file>` / `mgimind restore <file>` | gzip+tar всей директории данных. |
| `mgimind export [--format json\|md] [--output <dir>]` | Экспортировать памяти в файлы. |
| `mgimind import <obsidian\|markdown> <path> [--library <l>]` | Импорт папки markdown (рекурсивно, с дроблением). |
| `mgimind doctor [--fix]` | Health-check; `--fix` качает то, что отсутствует. |

## Конфигурация

Конфиг лежит в `~/mgimind/config.json`. Поля, влияющие на retrieval:

| Поле | По умолчанию | Смысл |
|---|---|---|
| `model_name` | `multilingual-e5-base` | Директория модели эмбеддингов под `models/`. |
| `vector_size` | `768` | Размерность эмбеддинга. Должна совпадать с моделью. |
| `pooling` | `mean` | `mean` (e5, MiniLM) или `cls` (некоторые XLM-R модели). |
| `uses_token_type_ids` | `false` | `true` для моделей BERT-семейства, `false` для XLM-R / e5. |
| `query_prefix` / `passage_prefix` | `query: ` / `passage: ` | e5 их требует; пустые для моделей, которые не требуют. |
| `rerank_enabled` | `true` | Реранкинг кросс-энкодером. См. языковую заметку ниже. |
| `rerank_model` | `bge-reranker-base` | Директория реранкера под `models/`. |
| `rerank_top_k` | `20` | Сколько кандидатов брать и реранкать перед возвратом `limit`. |
| `qdrant_port` | `6334` | gRPC-порт Qdrant. |
| `qdrant_api_key` | нет | Если задан, Qdrant стартует с ним и клиент аутентифицируется. |

## Языки и реранкер

Стек по умолчанию заточен под английский, потому что там ассистент сам
рассуждает лучше всего и модели сильнее всего. Это рекомендуемый сетап
для English-first проекта.

Несколько честных деталей:

- Эмбеддер (multilingual-e5-base) действительно мультиязычный. Английский
  запрос находит русскую заметку и наоборот, и поиск сам по себе хорошо
  работает между языками.
- Дефолтный реранкер (bge-reranker-base) заточен под английский. Он
  улучшает ранжирование на английском, но **снижает качество ранжирования
  на русском** (и других не-английских языках). Это единственное место,
  где дефолты предпочитают английский.
- **Если контент в основном на русском (или другом не-английском):** ставь
  `rerank_enabled = false`. Гибридный dense+sparse поиск сам по себе
  хорошо ранжирует эти языки; именно English-tuned реранкер их портит.
  Или подмени на сильный мультиязычный реранкер.
- Реранкинг стоит инференса кросс-энкодера на запрос — примерно одну-две
  секунды на 20 кандидатов на CPU-only коробке. Снижай `rerank_top_k`
  или выключай реранкинг для более быстрого поиска.

## Смена модели эмбеддингов

Смена модели обычно меняет размерность вектора, так что существующие
памяти нужно переэмбеддить:

1. Сначала backup: `mgimind backup ~/mgi-backup.tar.gz`.
2. Поставь `model_name`, `vector_size`, `pooling`, `uses_token_type_ids`
   и префиксы в `config.json` под новую модель.
3. `mgimind doctor --fix` чтобы скачать. Только дефолтные модели pinned;
   кастомная скачивается с предупреждением «integrity not verified», так
   что пин её SHA-256 в `integrity.rs`, если хочешь строгую проверку.
4. `mgimind migrate` чтобы переэмбеддить всё из сохранённого текста под
   новой моделью.

## Траблшутинг

- **«Model not found ... run doctor --fix»** — модели нет в
  `~/mgimind/models/`. Запусти `mgimind doctor --fix`.
- **«invalid expand shape» / inference errors** — обычно вход сильно за
  пределами 512 токенов. `add` дробит сам; если зовёшь библиотеку
  напрямую, дроби заранее.
- **Поиски медленные** — это реранкер на CPU. Снижай `rerank_top_k` или
  ставь `rerank_enabled = false`. Модели остаются тёплыми на всё время
  жизни процесса `mgimind mcp`, так что только первый лукап сессии
  платит за загрузку.
- **Инструмент падает сразу после установки** — запусти `mgimind doctor`
  (ассистент может позвать `mind_doctor`); он сообщает, что именно
  отсутствует (Qdrant не запущен, модель не скачана, ONNX Runtime
  отсутствует, файл карантинен AV) и `--fix` качает то, что может.
- **Предупреждение о несовпадении размерности** — векторы коллекции не
  совпадают с `vector_size`, обычно после смены модели. Переэмбедди
  через `mgimind migrate`.
- **Русские результаты странные** — поставь `rerank_enabled = false`
  (см. языковую заметку выше).

## Безопасность

- Загрузки проверяются на pinned SHA-256 (fail-closed) для ONNX Runtime
  (linux-x64), Qdrant и дефолтных моделей (e5 и реранкер). Другие
  платформы и кастомные модели предупреждают вместо слепого доверия.
- Qdrant биндится только на `127.0.0.1` и поддерживает API-ключ.
- Vault — AES-256-GCM с ключом, выведенным через Argon2id (параметры
  pinned, чтобы апгрейд библиотеки не залочил тебя снаружи). Только
  терминальный — мастер-пароль и расшифрованные секреты никогда не идут
  по MCP-каналу. MCP-инструменты `mind_vault_*` возвращают инструкции
  для терминала, не значение секрета.
- Записи файлов атомарны и директория-fsync'ится — краш оставляет
  старый файл или новый, никогда не битый.

Текущая версия: **1.0.x** (semver-стабильная с v1.0.0). Поверх
audit-лога 0.10.x и эфемерного viewer'а, карантина 0.11.x и
best-effort retrieval, viewer-волны 0.12.x, session liveness 0.13.x и

`MGIMIND_MODEL_VARIANT={cpu|gpu|auto}` и MCP-поверхность из 31 tool —
заморожены до 2.0.

Проект прошёл аудит кода на 27 проблем. **21 полностью пофикшена,
6 частично** — механизм отгружен, харднинг продолжается.

отдельности, включая пробелы (например, fact supersession ещё не
реализован). [`CHANGELOG.md`](CHANGELOG.md) хранит покомментарную

историю.

## Структура проекта

```
src/
  cli.rs         диспетчер команд и рендеринг вывода
  storage.rs     Qdrant: одна коллекция, гибридный поиск, history, stats, migrate, дробление
  embedder.rs    ONNX-эмбеддинг (модельно-зависимый пулинг, префиксы, cap 512 токенов)
  reranker.rs    реранкинг кросс-энкодером
  knowledge.rs   факты графа знаний
  procedure.rs   процедурная память: learn / recall / outcome
  ingest.rs      авто-извлечение и приём кандидатов
  relevance.rs   гейт релевантности v0.11 (длина, blacklist, decision markers, token novelty)
  consolidate.rs слияние дубликатов, отчёт по cold-записям
  md_reconcile.rs md-импорт как reconcile с «md wins»
  audit.rs       append-only audit-лог на каждую мутацию хранилища
  viewer.rs      эфемерный локальный HTTP-viewer (axum, статический фронтенд встроен)
  session.rs     сессионные файлы на ассистента
  vault.rs       зашифрованный vault секретов (только терминал)
  mcp.rs         MCP-сервер по stdio (hand-rolled JSON-RPC; модели тёплые в процессе)
  config.rs      конфигурация
  integrity.rs   pinned SHA-256 хеши для загрузок
  util.rs        атомарные записи, верифицированные загрузки
tests/
  cli_integration.rs   black-box тесты против реального Qdrant (CLI + MCP round-trip)
```
